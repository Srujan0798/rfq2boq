06

Key Learnings

🔬 Technical

  • Data > Architecture
    Synthetic 99% F1 → real 43% F1. The data was the bottleneck, not the model.
  • Honest Evaluation Matters
    Self-comparison gives fake 100%. Independent gold is the only valid metric.
  • PDF >> Excel
    Tables in PDFs are fragile — merged cells, multi-line, split quantities.
  • Pattern NER Wins on Small Data
    With 20 annotations, regex + gazetteer beats BERT-LoRA.

⚙️ Process

  • One Agent at a Time
    Parallel agents on one repo = deleted files, chaos, lost work.
  • Commit Before Next
    Every fix gets committed and verified before the next task.
  • Scope Guard
    Saying "no" to out-of-scope features (SaaS, papers, patents) kept us focused.
07

Challenges Faced

  • 1. The Synthetic-to-Real Gap
    Model trained on 300 synthetic PDFs (regex-generated from research papers) scored 99% F1. On real tenders: MATERIAL F1 = 0.0. The model learned research prose, not construction language.
  • 2. PDF Table Fragility
    Merged cells, multi-line cells, split-quantity columns in GeM-style PDFs. Text classifiers miss digit-only columns.
  • 3. The "Fake 100%" Trap
    Earlier iterations modified gold files to match pipeline output. We caught this, implemented anti-cheat rules, and now all numbers are independently verified.
