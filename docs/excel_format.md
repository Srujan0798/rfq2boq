# CPWD Excel BOQ Format

Documentation for the CPWD-format Excel Bill of Quantities output generated by `src/export/excel_generator.py`.

---

## Output Structure

### Header (Rows 1–8)
- **Row 1:** Title "BILL OF QUANTITIES (CPWD FORMAT)" — bold, blue (#1F4E78), 14pt
- **Rows 2–6:** Project metadata — Project Name, Location, Date, RFQ Reference, Contractor
- **Row 8:** Column headers (row 9): S.No | DSR Code | Description | Unit | Quantity | Rate (₹) | Amount (₹) | Notes

### BOQ Data (Rows 10+)
Items grouped by trade section:
- Excavation
- Concrete
- Brickwork
- Plaster
- Flooring
- Steel
- Woodwork
- Painting
- Plumbing
- Electrical
- Waterproofing
- Finishing

Each section ends with a subtotal row (light blue fill).

### Totals
- **GRAND TOTAL** row — dark fill (#2E4057), white text, double border
- **Add GST 18%** row
- **NET TOTAL (incl. GST)** row — bold, larger font

### Footer
- Notes section (4 standard notes)
- Signature blocks: Prepared by / Checked by / Approved by

---

## Column Details

| Col | Header | Width | Formatting |
|-----|--------|-------|------------|
| A | S.No | 6 | Center-aligned integer |
| B | DSR Code | 11 | Text (preserved as string, not date) |
| C | Description | 42 | Wrap text, left-aligned |
| D | Unit | 8 | Center-aligned |
| E | Quantity | 10 | Indian format `#,##,##0.00` |
| F | Rate (₹) | 13 | Indian format `₹#,##,##0.00` |
| G | Amount (₹) | 14 | Indian format `₹#,##,##0.00` |
| H | Notes | 20 | Center-aligned |

---

## Indian Number Formatting

Numbers use lakhs separator format: `₹#,##,##0.00`
- 1,23,456.00 (not 123,456.00)
- ₹ symbol prefix via Excel number format string

---

## DSR Code Handling

- DSR codes like "2.1.1" are stored as text to prevent Excel from parsing as date
- When a BOQ item description matches a CPWD DSR entry, the official DSR code and rate are used
- Unmatched items show blank DSR code with "rate estimated" note

---

## Formula Usage

All totals use Excel SUM formulas so they auto-update when quantities/rates are edited:
- Subtotals: `=SUM(G[start]:G[end])`
- Grand Total: `=SUM(G[items_start]:G[items_end])`
- GST: `=G[grand_total]*0.18`
- Net Total: `=G[grand_total]+G[gst_row]`

---

## Styling

- Header row: Bold, white text on blue (#1F4E78) fill
- Trade subtotals: Bold, light blue (#D9E1F2) fill
- Grand total: Bold white on dark (#2E4057) fill with double bottom border
- All cells: thin border all around

## Usage

```python
from src.export.excel_generator import CPWDExcelGenerator

gen = CPWDExcelGenerator()
gen.export(
    boq_items=[
        {"material": "Earthwork", "quantity": 100, "unit": "cum", "rate": 245.5}
    ],
    output_path="output/boq.xlsx",
    project_metadata={
        "project": "Project Name",
        "location": "Delhi",
        "reference": "REF/001",
        "contractor": "ABC Corp"
    }
)
```

Compatible with `src.domain.models.BoqRow` via `_as_dict()` conversion.
